Thinking like a hacker:

Security considerations for
High-Fidelity Web Archives

Ilya Kreymer, Webrecorder

Jack Cushman, Perma.cc

Why is security a concern?

  • Web Archives just collection of old pages
  • High-Fidelity web archives run untrusted web software
  • Live site is "safe", so nothing to worry about
  • Web archive replay can pose new security risks

Possible security threats

  • Unauthorized capture
  • Manipulating capture or replay
  • Threats to users data
  • Threats to credibility of a web archive

Threat: Unauthorized capture

  • Capture system may have priveleged access to local network
  • Possible to capture local private resources, into a public archive

Mitigation: Network Filtering

  • Don't allow capture of local ip ranges
  • Run capture system in isolated sandbox

Concern: URL rewriting

  • Untrusted content accessed from web archive domain
  • Separate domains:
    • http://example-domain-A.com/
    • http://example-domain-B.com/
  • Rewritten replay, same domain:
    • http://example.com/http://example-domain-A.com/
    • http://example.com/http://example-domain-B.com/
  • Standard cross-domain protections do not apply!

Threat: Cross-Domain Cookie/Storage Access

  • One site can access cookies or local storage of another
  • More concern on capture, but also on replay

Possible Mitigation: Cookie Rewriting

  • Rewriting system isolates cookies and local storage per host
  • Domain wildcard cookies?
  • Not foolproof
