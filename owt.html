oldweb.today

Web Archiving + Emulation

Ilya Kreymer & Dragan Espenschied

Architecture and Concepts

Layered System, “Separation of concerns”

  • Web archives provide web pages (Web archives/Memento)
  • Web pages running in Browsers (Emulation)
  • Browsers running in Emulators (Emulation)
  • Emulators running in Containers (Emulation)
  • Containers running in a VM (Amazon EC2)

Benefits of emulated browsers

  • Complex dynamic site issues (Javascript, cookies, etc…)
  • <blink> tag
  • Legacy software dependencies: Flash, Java, Shockwave, etc…
  • Live Leaks impossible
  • Effectivity of emulation approach: a dozen containers for millions of web pages

Emulators: What is being emulated?

Old OSes and Browsers

  • Browser is the platform for web sites (more important than OS itself)
  • Differences from other emulators:
    • Gaming, other software emulators require exact timing — not needed here
    • Latency OK
    • New challenges: networked emulators

Web emulation, not internet emulation

  • HTTP protocol has not changed much since original inception
  • HTTP proxy mode support (and FTP, SOCKS) makes emulation + web archiving

HTTP Proxy

  • Easy mechanism for serving archived web content instead of original
  • Difficult to configure manually
  • But easy if done if configured in a container

Containers (Docker)

  • Containers are not VMs
  • Containers provide assembled code+data that can be deployed rapidly
  • Docker provides scheduling, resources sharing and limiting, networking and security
  • Perfect for launching multiple instances of complex systems on same machine (VM)
  • oldweb.today supports roughly 5-6 browser containers per EC2 CPU

Memento Aggregation

  • Grab data from multiple archives simultaneously
  • Using memento timegate for finding closest match
  • Builds on work with Memento Reconstruct
  • Memento limitations: Only archives supporting unaltered content are considered

Cultural Impact

The web as performative media

oldweb.today makes the variability of the web tangible.

The web has, since the advent of client-side scripting in 1995, been a software delivery system rather than a document delivery system.
